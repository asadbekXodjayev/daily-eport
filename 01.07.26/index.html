✓ Главное за день
Ключевой фикс — SARB-DISP-No. Корень проблемы был в резолвере получателя оценки: для менеджера груза он «дотягивался» до negotiation_dispatcher_id — это ретранслирующий третий диспетчер, а не участник рейса. Когда владелец груза назначает водителя напрямую (без менеджера водителя), резолвер всё равно возвращал чей-то id → всплывало лишнее окно оценки и бэкенд отклонял отправку. Теперь резолвер для обеих ролей самодостаточен: возвращает только настоящего участника рейса или null (окно не показывается вовсе).
⚠ Что требует ручной проверки QA
  • SARB-DISP-No, «правильный» сценарий — когда менеджер водителя действительно участвовал (принял оффер, у рейса выставлен driver_manager_id): окно «Оценка менеджера водителя» должно по-прежнему появляться и отправляться без ошибки. Требует клика под авторизацией на завершённом рейсе с менеджером водителя.
  • Список «Все грузы» — визуально показатель «машины» подтверждён на рендере реального компонента (см. скриншоты); финальный пиксельный клик по live-странице под авторизацией с реальными данными рекомендуется в staging.

01.07.2026 · 17:08

SARB-DISP-No: ошибочная оценка менеджера водителя logic bug

  • Симптом: менеджер груза завершает рейс, оценивает водителя — и следом всплывает окно «Оценка менеджера водителя», хотя менеджера водителя в рейсе не было. Отправка → POST /v1/dispatchers/dispatchers/{id}/rating → 400 «получатель оценки не совпадает с этим завершённым рейсом».
  • Причина (2 связанных дефекта): (1) резолвер получателя падал в negotiation_dispatcher_id (ретранслятор), затем в generic-блок «любой другой диспетчер»; (2) авто-подсказка на TripsPage открывала окно без проверки участия менеджера водителя.
  • Исправление: обе ролевые ветки резолвера самодостаточны (только явные driver-/cargo-manager-сигналы, иначе null); guard cargoManagerHasDriverManagerToRate на TripsPage. В ходе code-review восстановлен offer.counterparty_id для DM→CM (единственный легитимный сигнал старого generic-блока), ретранслятор при этом исключён.
  • Проверки: +4 unit-теста (всего 9 в наборе резолвера), tsc -b/vite build exit 0, 572/572 тестов, verify — рендер модулей через Vite.
01.07.2026 · 17:30

«Все грузы»: показатель «машины» n/n → только остаток ui bug
