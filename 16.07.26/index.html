0 Первопричина FIX

Объект Driver в API несёт два статусных поля: сырое, nullable account_status и вычисляемое effective_status = COALESCE(NULLIF(TRIM(account_status),''),'active'). По контракту: «account_status null/пустой — водитель функционально активен». Фронт рендерил сырое поле — поэтому большинство водителей (у которых статус никогда не трогали) показывались как «No data», и по ним же падал activate: null-водитель уже активен, /moderation/inactive принимает null как источник, а /moderation/activate — нет. Отсюда «деактивируй, потом активируй — работает».

1 Колонка «Account» — эффективный статус FIX

Теперь колонка показывает эффективный статус (null → Active). Никаких «No data» — они остались только у настоящего пустого рейтинга. Значение используется и в сортировке, фильтрах и признаке isBlocked.

/en/admin/drivers — Account: Active / Blocked / Inactive по каждой строке.
Русская локаль — та же колонка.

2 Действия строки: только View · Edit · Delete UI

Кнопка «Send to moderation» убрана из колонки действий. Вся модерация теперь живёт внутри модалки редактирования.

Водители: View · Edit · Delete.
Диспетчеры: View · Edit · Delete.

3 Account / KYC / Moderation — редактируются в модалке FEATURE

Новый блок в модалке редактирования водителя делает все три статусные оси управляемыми:

  • Account — activate / deactivate / block / unblock (дропдаун Account status, гейтится по статусу).
  • KYC — Accept / Reject (с причиной), эндпоинты /kyc/accept, /kyc/reject.
  • Moderation (допуск) — Accept / Reject (с причиной) / Send to moderation.
Модалка «Edit driver» — блок Account / KYC / Moderation вверху. Лейбл дропдауна — «Account status», не «Moderation».

4 Activate больше не падает на «No data» FIX

Дропдаун «Account status» для уже активного водителя предлагает только Deactivate и Block — Activate скрыт (раньше именно он давал 400 на null-статусе). Кнопка «Send to moderation» задизейблена, пока статус модерации ≠ Rejected: у водителя resubmit строго rejected → pending (иначе 400 driver_not_rejected).

Активный водитель → в меню только Deactivate / Block. «Send to moderation» серая (статус Pending).

5 Диспетчеры — тот же паттерн FIX

Ось жизненного цикла диспетчера — moderation_status (а не account_status, который у диспетчера захардкожен в «active»). Секция Moderation переехала в модалку; isBlocked теперь читается из moderation_status. Resubmit у диспетчера работает из любого статуса, поэтому остаётся в дропдауне модалки.

/en/admin/dispatchers — Status + Moderation заполнены.
Модалка «Edit dispatcher» — секция Moderation с дропдауном.
Дропдаун лайфсайкла диспетчера.
Русская локаль.
