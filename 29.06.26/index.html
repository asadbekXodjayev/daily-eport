Создание груза: выбор города ничего не находил при кросс-алфавитном поиске (напр. RU/Краснодар)

  • Причина: CitySelect использует серверный поиск, но затем повторно фильтровал результаты клиентом через smartSort, который отбрасывал записи со счётом <1. Кириллический запрос против латинского лейбла → −1 → отброшено. Ломало любой кросс-алфавитный/транслит-поиск.
  • Исправление: добавлен smartSortKeepAll — переупорядочивает по релевантности, но никогда не отбрасывает. CountrySelect оставлен на smartSort (там фильтрация локального списка корректна).
29.06.2026 · 11:55

Создание груза: выбор города показывал несколько одинаковых на вид городов

  • Причина: /v1/reference/cities возвращает много разных населённых пунктов с одинаковым именем (у каждого свой code/координаты). Лейбл был только city.name → визуально неразличимы. Это не дубликаты — схлопывание по имени рисковало бы отправкой груза не туда.
  • Исправление: новый buildCitySelectOptions: дедуп только точных повторов по code, выводит регион (region_name) вторичным признаком, а для редких совпадений имя+регион добавляет уникальный суффикс. Двухстрочная опция (имя + регион), контрол остаётся именем. Добавлен a11y-безопасный токен цвета подписи.
29.06.2026 · 12:05

Редактирование груза: только что загруженное фото не уходило в payload — тихая потеря данных критично

  • Причина: buildCargoUpdatePayload собирает payload по белому списку и никогда не копировал в него photos (в CREATE — копировал). Фото грузилось, попадало в форму, но отбрасывалось на этапе payload → к грузу не привязывалось. Старый тест даже кодировал этот баг.
  • Исправление: payload теперь шлёт photos, но только когда набор фото гарантированно загружен (includePhotos по успешному GET — защёлка-ref). Это закрыло и опасный сценарий: при сбое GET /cargo/{id}/photos наивный «всегда слать фото» затёр бы все серверные изображения → теперь при сбое фото опускается, бэкенд сохраняет существующие.
  • Каверзно: найдено и закрыто адверсариальным ревью (2 агента) — регрессия затирания при сбое загрузки + латентный близнец в CargoForm.
